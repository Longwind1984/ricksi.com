
# 招聘官批判性分析：ricksi.com（司豪杰 Rick Si 个人工作台）

> 评审身份：AI 产品总监 / AI PM 岗位招聘官。评审对象：`rick-homepage` 站点现状 + 已知 v3 迭代计划（3D 图谱、知识库扩至 ~620 篇、AI 共创阅读区、token 重估算）。面试窗口：2026-06-17。
> 评审原则：每一刀落在具体证据上。引用均来自站点源码、文案原文与数据文件。

---

## 1. 30 秒第一印象：一个漂亮的仪表盘，和一个不存在的候选人

打开页面 30 秒，我看到的是：玻璃质感、雪山云海、一个 8 位数的大数字「39,598,302」、热力图、力导向图谱。审美在线，工程在线。**传递的信息是「这个人重度使用 Claude，且有品味」。**

但 30 秒内我找不到招聘决策需要的任何一项：

- **候选人叫什么**——名字只在浏览器标签页 `<title>` 和 JSON-LD 里，hero 区从头到尾没有出现「司豪杰」三个字。品牌位写的是 `Rick's Home Page`，一个默认占位符级别的命名。
- **他做过什么**——「互联网产品经理」五个字是全站对过往职业生涯的唯一描述。几年经验？什么行业？什么规模的产品？什么业绩？零信息。转型叙事有「to」没有「from」。
- **我能拿到什么**——导航栏「下载简历」按钮点击后滚动到页脚，页脚的「下载简历 PDF」是 `href="#"` 加 `noop` class，JS 里 `e.preventDefault()`——**点击后什么都不发生，没有提示，没有降级**。招聘官在这个站上意图最强的一次点击，被无声吞掉。README 自己都写着「待替换素材：『下载简历』按钮的 PDF」。

30 秒结论：这是一个「AI 爱好者的精美仪表盘」，不是一个「我应该给面试机会的 PM」的证据包。

## 2. 叙事与定位：标题写着 AI 产品经理，正文在演工程师

**Hero 主张经不起站内证据的反查。** H1 是「把产品判断，变成能跑起来的东西。」——但 5 个项目里 3 个（Slash Goal、行程规划套件、WeatherLens）`href: '#'`，点击无反应；剩下 2 个链向 GitHub 仓库而非可运行的东西。项目区副标更狠：「每个判断背后，都有一个能跑起来的项目」——**全站唯一真正「能跑起来」给我看的东西，是这个主页本身**。主张和证据之间的落差，恰好是 PM 面试里最忌讳的那种。

**身份表述自相矛盾。** JSON-LD 里 `jobTitle: 'AI 产品经理'`，hero 副标写「正在转向 AI 产品方向」——元数据里宣称职位，文案里对冲措辞。要么你已经是，要么你正在成为，挑一个。

**动词暴露定位漂移。** 副标自述「读源码、建工具、写思考、养知识库」——四个动词全是工程师/学者的动词。全站找不到「用户、增长、取舍、上线、复盘」这些 PM 动词对应的任何证据。这个站把候选人定位成了「有品味的初级 AI 工程师」，而 AI PM 岗位真正稀缺的组合是「成熟的产品判断 × AI 流畅度」——本站对后者超额供给，对前者供给为零。

**我该记住的一句话是什么？** 看完全站，我能记住的是「他用 Claude 用得非常凶」。这不是一个 hiring signal，这是一个消费记录。

## 3. 项目模块逐个拆（5/5）

通病先说：**五个项目没有一个交代「我的角色 / 关键决策 / 取舍 / 结果」四要素**，全部是功能与工作量罗列。逐个看：

### 3.1 Slash Goal 复刻（featured，无链接）
文案：「在 Claude Code 原生 Slash 命令发布之前，研读 Codex 开源代码并完整复刻其斜杠命令交互。能读懂工程实现，是与研发对话的底气。」
- 我会追问：复刻交互的过程中你做了什么**产品**决策？「在原生功能发布之前」这个时间主张如何验证？代码在哪？
- 现有描述一个都答不了：`href: '#'`，无仓库、无截图（README 承认封面是「设计 SVG」占位）、无产出。
- 更糟的是自我定位：「能读懂工程实现，是与研发对话的底气」——这句话把项目的全部价值归结为「我能听懂工程师说话」。这是把 PM 的**及格线**当成**亮点**来卖。复刻一个已有交互，没有用户、没有判断、没有取舍，不该作为 featured 第一位。

### 3.2 行程规划套件（featured，无链接）
文案：「把『调研 → 编排 → 优化』的模糊需求拆成一套日常在用的规划工具，独立走完想法到可用的全程。」
- 「日常在用」——用户 n=1（他自己）。我会追问：除你之外有人用过吗？「可用」的判据是什么？skill 的失败率？
- 知识库里其实埋着这个项目最有价值的素材：`trip-structure-skill.md` 写了「中间一度滑向 over-design，被 Rick 拉回收敛；section 3、4 删除」——**这才是 PM 含金量（识别过度设计并砍需求）**，但项目卡片上一个字没提，宝贵的决策证据埋在 141 篇笔记的第 138 篇里。

### 3.3 博物馆互动导览（featured，链 GitHub）
文案：「可互动演示的导览小产品……可以直接上手玩。不做覆盖型百科，只答『它为什么在这里』。」
- 「只答『它为什么在这里』」是全站项目文案里**唯一一句真正的产品定位判断**，值得肯定。
- 但「可以直接上手玩」点过去是一个 GitHub 代码仓库。承诺了 demo，交付了源码——对非技术面试官等于零；对技术面试官等于「他不知道 GitHub Pages 要五分钟」。

### 3.4 MuseumCollect 智能体协作案例（非 featured，收在折叠区）
文案：「8 个有边界的 AI agent 跑 5 个 sprint：converged demo、11 份多视角 audit、2 轮 cold-audit 拷问与修复、万字 case-study 与 6 份商业 PM 产出——把 AI 的约束当作产品设计原则的实证。」
- 这是全站**唯一有仓库、有 case study、有方法论主张**的项目，却被收在「展开全部」的折叠区里，让三个无链接项目占 featured 位。策展判断本身就是减分项。
- 文案是数字轰炸：8、5、11、2、万字、6——**全部是投入量指标，零产出指标**。11 份 audit 发现了什么？2 轮 cold-audit 推翻了哪个设计？「6 份商业 PM 产出」是什么、给谁用、被验证过吗？「converged demo」这种中英夹杂的内部黑话，面试官第一次读根本不知所云。
- 我会追问的就一个问题：「把 AI 的约束当作产品设计原则」——举一条约束、一条由它推出的设计原则、一个反例。现在的描述完全撑不起这句口号。

### 3.5 WeatherLens 天气透镜（无链接）
文案：「MapLibre GL + Open-Meteo 天气瓦片的端到端技术验证：123 个变量、93 个时间步的实况预报图层，附 API 验证报告与技术决策文档。」
- 这是一份纯技术 PoC 简报。123 个变量、93 个时间步——为了什么产品问题？给谁用？验证的结论是「做」还是「不做」？没有问题陈述的技术验证，对 PM 候选人是负资产：它强化「他在做工程师的事」的印象。`href: '#'`，连那份「验证报告与决策文档」都看不到。

**项目区总判**：该换讲法的（MuseumCollect 提为第一、用决策-取舍-结论重写），该降级合并的（Slash Goal + WeatherLens 合成一行「技术验证练习」），该补 demo 的（博物馆）。现状是把最弱的放最前、最强的藏起来、所有的都用工作量代替判断。

## 4. 数据工作台：把消费记录当作品集

「不是装饰——这是我每天的真实用量」——这句副标是全站最大胆也最危险的一句话。拆开看：

- **「每天」站不住。** `usage.json` 的 `method` 字段写得很诚实：`real_window: '2026-05-27 起'`，`estimated_days: 224`——时间序列 240 天里 **224 天（93%）是估算值**，真实日志只有约两周。页面上「早期估算占 31%」的标注是按 token 量算的，数字没撒谎，但叙事撒了：所谓「活动史回溯到 2022 年」的用量曲线，是用活跃度倒推出来的拟合，不是记录。
- **累计 2.0B 里 1.2B 发生在最近一周。** `week: '1.2B'`，`cumulative: '2.0B'`——60% 的「累计」是过去七天产生的，其中 6 月 7 日单日 809.68M（`days7` 周日项）。任何看过 Claude Code 账单的面试官都能猜到：那一天是批量生成知识库内容。**这个计数器测量的，主要是「为这个网站生产证据」这件事本身**——proof-of-work 的数据由生产 proof-of-work 的工作贡献，完美的循环论证。
- **「含缓存读写」是数量级级别的水分。** 以 2026-06-06 为例：opus 当日 total 153.7M，其中 cache read 141.5M（92%），真正的输出 token 只有 1.19M（0.8%）。脚注里四个小字「含缓存读写」技术上免责了，但 2.0B 这个数字对懂行的人会瞬间缩水一到两个数量级，且会留下「他选了最大的那个数来展示」的印象。
- **提交数是噪音。** 「154 commits / 连续 17 天 / 6 个仓库」——154 个提交对任何一家科技公司的招聘官来说不是信号，把它和 token 数并列展示，反而暴露了「能展示的硬指标只有这些」。
- **最锋利的反问**：「Token 用量证明你用得多。**如果明天 Anthropic 封了你的账号，这个站还剩下什么能证明你？**」用量大可以是杠杆高，也可以是挥霍、是不会写 prompt、是让 Opus（占比 81%）干 Haiku 的活。一个 AI PM 连自己的 token 都不做成本分层路由，「推理成本控制」那篇 m209 笔记是写给谁看的？——除非他能当场把 81% Opus 讲成一个有意识的质量/成本权衡，否则这个模块是给面试官递刀。
- **一个设计层面的诚信地雷**：`index.astro` 的回退逻辑是数据缺失时静默使用 `SAMPLE_USAGE`（today: 1,247,832 等编造数字），而「不是装饰——这是我每天的真实用量」的标题**照常渲染**，只是少了更新时间戳。当前因为 JSON 已入库不会触发，但这个「失败时撒谎」的默认设计，和站点标榜的诚实标注精神相悖。

## 5. 知识库与图谱：141 篇里 81% 诞生于同一个六天窗口

`grep created` 的结果：141 篇笔记中 **115 篇（81.5%）created 落在 2026-05-13 至 05-18 六天之内**。首页文案写「双链生长中」——这不是生长，这是浇筑。666 条双链里大量是模板化的「关联节点」清单（每篇笔记尾部例行链回总索引），图谱拓扑是轮辐状而非涌现状。

**内容质量是双峰分布**，这比单纯灌水更需要小心处理：

- **好的一端**：`a01-agent-概念史与语义流变.md`、`a03-react.md` 有真实判断密度——a03 的「四个已被验证的弱点」每条对应工程预算项，a01 的修订日志（R1→R4）展示了引入 Karpathy、Weizenbaum 反方框架的对抗性迭代。这是站内最接近「思考过程证据」的东西。
- **差的一端**：`attention.md` 是任何教程都有的 QKV 三行解释加一张 GQA 表——面试官读完的反应是「这是 Claude 的知识，不是你的」。`skill-系统的本质.md` 尾部挂着「## 衍生对话存档 - 来源对话」的死桩文本；全库 **44 篇带「衍生对话存档」残桩、18 篇带「待补充」占位**——发布管线把内部 TODO 状态原样倒进了公开站点。
- **暴露 AI 代笔结构的硬证据**：`anthropic.md` 有「对 Rick 的价值」「职业判断锚点」章节，a01 §8 标题直接是「严肃定义建议（**写给 Rick 转型期使用**）」——笔记用第三人称称呼库主，等于公开承认「这是 AI 写给我的教材」。a01 §8 还原样贴着一大段未经编辑的语音转写（「那其其实区别于」「==刚才 opus 澄清了一下」），`ai概念滥用反思.md` 的一句话定义里保留着「直接斥之为『他妈生硬，愚蠢』」——**求职站点上公开发布的脏话**。`claude.md` 的自动证据池泄漏了私人日记标题：「20260424-刚才在费城的青旅」「20260403-哥伦比亚 🇨🇴 54 天」——招聘官顺手就推断出长期旅行 gap，这个叙事应该由候选人主动讲，而不是被爬虫式证据池出卖。
- **招聘官抽读一篇的真实心理过程**：点开 `attention.md` → 「通用教科书内容」→ 点开 `anthropic.md` → 读到「对 Rick 的价值」→ 意识到整库是 AI 生成的学习材料 → **回头重新评估首页那个「141 篇笔记」的数字：它从『知识资产』降级为『token 消费的另一种计数法』**。

**对 v3 扩到 620 篇（含 475 篇 AI 共创全量发布）的预判**：如果 141 篇已经触发上述降级，475 篇同模板批量笔记全量上线等于把红旗升到旗杆顶。「全量发布」恰恰反掉了 PM 最该展示的能力——策展与取舍。620 篇没人读得完，但任何一篇被抽中的 AI 味文本都会代表整库被定罪。数量在这里是负杠杆。

**3D 图谱（NASA Eyes 风格）的炫技/实质边界**：2D 图谱已经支持搜索、过滤、focus 深链——功能完备。3D 增加的是 wow，删除的是可读性（标签遮挡、移动端性能、轮辐拓扑在 3D 里只是更立体的毛线球）。在面试前一周，把工程预算花在 3D 渲染而不是简历 PDF 和项目链接上，本身就是一个可以被面试官引用的**优先级判断反面案例**。候选人自己的全局协议都写着「第二个真实用例出现前，不引入抽象层」——v3 在违反他自己的架构信条。

## 6. 阅读区：唯一的全人类信号，即将被 AI 稀释

真实数据（`reading.json`，weread 官方网关）：藏书 400、**读完 24（6%）**、在读 72、划线与想法 3,321 条、阅读 552 天、累计 213 小时（= 日均 23 分钟）。书架前排：《人的行动》0%、三本青铜器书全部 0%、《政治的神学》3%、《美国生活中的反智主义》5%，最深的是《论美国的民主》57%。

- **对求职的真实贡献**：托克维尔、施米特、霍夫施塔特 + 青铜器——这个书架证明智识趣味和人文纵深（也确实支撑了博物馆导览项目的领域兴趣），但**架上看不到一本 AI / 产品 / 技术书**，与「输入决定输出——书架也是工作台的一部分」的副标直接断裂：如果输入决定输出，那这些输入对应的输出是 AI PM 吗？
- 「藏书 400 / 读完 24」诚实得近乎自伤。招聘官的直觉翻译是「收藏家，不是完成者」。3,321 条划线想法才是这个区块真正的金矿——却一条都没展示，展示的是一排 0% 进度的封面。
- **「AI 共创 ePub」叙事的强与弱**：强——它是「用 AI 生产个性化学习材料」工作流的独特物证，没几个候选人有。弱——它一句话就能被击穿：「**所以你读的书也是 AI 写的？**」阅读区目前是全站唯一 100% 人类行为的数据源；把 AI 生成的 ePub 混进来，等于亲手污染最后一个对照组。如果一定要做，必须物理分区并标注成「方法论展示」，绝不能和真实书架共用「阅读」统计。

## 7. 设计与体验：工程值得尊敬，但替错了优先级

先给应得的：feDisplacementMap 折射、三档降级、LQIP 占位、View Transitions、⌘K 面板——执行水准高于绝大多数个人站。「Liquid Glass 是不是在做设计师的事」本身不是问题，**问题是它和占位符共存**：当折射滤镜的位移贴图都手工调过、而简历按钮是 `href="#"` 时，传递的不是「全栈品味」，是「优先级失控」。

具体粗糙点：
1. 简历双重死链（见 §1），且 noop 点击零反馈。
2. 三个无链接项目行保留「→」箭头视觉暗示可点，点了无声失败。
3. 「查看全部 1 篇 →」——博客只有一篇文章时，这个按钮是自嘲。区块副标「做的时候想清楚，写下来才算数」按它自己的标准：全站几乎什么都不算数。
4. 唯一一篇博文以「草稿说明：本文由 Claude 按全程协作记录起草，**待我修订后定稿**。——Rick」开头——候选人公开承认连自己唯一的写作样本都没改完。
5. 同一篇博文里硬编码「今天 90M+，本周 1.1B」，而页面实时组件显示今天 39.6M——**主打「活数据」的站点，散文和仪表盘互相打架**。
6. 页脚裸晒手机号 `+86 180 0600 4674` 带 `tel:` 链接——爬虫和骚扰电话的自助餐，且与知识库严格的隐私排除管线（excludeClusters）形成讽刺对照。

## 8. 致命缺失清单（「要不要给面试」决策点上缺的东西）

按对决策的阻塞程度排序：
1. **简历 PDF**——占位符。没有简历的求职站点是装置艺术。
2. **过往工作经历**——公司、年限、行业、产品规模、业绩数字，全部为零。「互联网 PM → AI PM」的「互联网 PM」部分是空集。
3. **任何一个可量化的结果**——全站没有一个用户数、留存、转化、收入、效率提升数字，包括 AI 项目在内。
4. **案例深度复盘页**——MuseumCollect 的「万字 case-study」存在 GitHub 仓库里，站内没有任何一个项目有落地页。面试官不会去仓库里考古。
5. **可玩的 demo**——文案承诺「可以直接上手玩」，全站零个嵌入式 demo。
6. **LinkedIn / 社交职业身份**——`sameAs` 只有 GitHub。
