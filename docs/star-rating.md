# 星图评级体系（前沿追踪）

> 给每条追踪到的前沿动态一套**多维**评级，替代单一分数 / SABCD。
> 完整 Agent 规约见 `星图评级体系-Agent规约-v1.md`（由 Claude Chat 拟）；本文是**落地版**——
> 记录我们实际实现了规约的哪一部分、怎么实现、为什么这样裁剪。

## 第一性原理：视亮度 ≠ 绝对亮度

策展人区别于算法热榜的全部价值，在于把「当下多大动静」和「真实多重要」解耦：

- **深空**不亮却重，**流星**很亮却空，**黑洞**不亮却主宰。
- 单轴分级（SABCD）把这两件事压成一条线；星图不压。

> 两条独立坐标：**事件量级**（每条动态的星类，含顶档「奇点」）与**信源量级**（人/机构的定性，含顶档「北极星」）。两者互不换算——一条普通推文也可能出自北极星级的人。

## 三个维度（采集端逐条打分，`collect-frontier` schema）

| 字段 | 含义 | 取值 |
|---|---|---|
| `apparent` | **声量 / 视亮度**：当下动静多大 | 1（几乎无声）– 5（出圈刷屏） |
| `absolute` | **分量 / 绝对亮度**：真实重要性，与热度无关 | 1（无实质）– 5（改写范式） |
| `gravity` | **隐体**：本体完全不可观测，重量只能从第三方连锁反应反推 | 布尔，极罕见 |
| `periodic` | 周期性回归的话题潮 | 布尔，少见 |
| `singularity` | **奇点**：技术演进史的分水岭（Attention 论文 / GPT-3.5 / DeepSeek R1 级） | 布尔，日常抓取几乎为 false |

派生：`hype_gap = apparent − absolute`。负值大 = **被低估**（深空，策展招牌）；正值大 = 声量盖过分量。

## 星类映射（确定性，`src/lib/frontier-ui.mjs` 的 `starOf`）

采集端只存维度分；星类是维度的**确定性函数**——规则单点在 `starOf`，改规则不必重抓数据。

| 条件 | 星类 | 象限语义 |
|---|---|---|
| `singularity` | 奇点 Singularity | **事件级最高**：重定义时代的分水岭 |
| `gravity` | 黑洞 Black Hole | 看不见却主宰 |
| `periodic` 且分量≤3 | 彗星 Comet | 周期回归 |
| 分量≥4 且声量≥4 | 超新星 Supernova | 高分量 + 出圈 |
| 分量≥4 且声量≤3 | 深空 Deep Field | **高分量却低声量 = 被低估** |
| 分量=3 且声量≥4 | 新星 Nova | 中分量 + 热闹 |
| 分量=3 且声量≤3 | 微光 Faint Glow | 日常有料 |
| 分量=2 且声量≥3 | 流星 Meteor | **低分量 + 高声量 = 炒作** |
| 分量=2 且声量≤2 | 微光 | 边角 |
| 分量=1 且声量≥3 | 流星 | 纯噪声爆点 |
| 分量=1 且声量≤2 | 星尘 Cosmic Dust | 背景水文 |

排序用分量主导（`starRank`，法则：亮≠重，分量主导）。

## 三层落地（规约 13 类如何匹配「逐条」管线）

规约有 13 类，但采集是**逐条**调 claude。按「评的是什么」分三层，不硬塞：

1. **条目级（claude 逐条自动）**：上表的 EVENT 类 + 黑洞 + 罕见的奇点。日常流主体。
2. **源级（config 人工标，`people[].constellation` / `topics[].constellation`）**：北极星 / 猎户座 / 星辰 / 行星——「人/机构」的定性，属档案，不让单条动态去判（北极星 = 顶档，重要的人与实验室）。
3. **场级（星云，v1 暂缓自动化）**：需跨时间聚合，逐条评级必然通胀乱命中；保留概念，留作未来手工策展位。

## 防通胀（实现要点）

- **顶档稀缺**：奇点 / 超新星 / 黑洞 命中须在 `rationale` 给理由；prompt 里 `gravity`/`singularity` 措辞极严（任何公开可读的内容一律 `gravity=false`；奇点仅限重定义时代的分水岭）——曾因措辞太松导致黑洞通胀，已收紧。
- **向下取整**：`absolute` 拿不准取低；在「炒作」与「实质」间犹豫，按实质打 `absolute`、把热度交给 `apparent`。
- **存量迁移**：`scripts/migrate-frontier-rating.mjs` 调 ephemeral Codex 仅评级补存量三维分（不碰已发布内容），口径与新采集统一。
