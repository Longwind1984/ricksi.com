# _自我民族志系统化专题·总览

> 本文件是 **0423 自我民族志系统化专题** 的 MOC（地图式入口）。专题对象不是"AI"这个领域，而是 **Rick 本人作为 AI 极端 power user 的使用模式**——这是它独一无二、没有竞争者的根，也是它最容易滑进自我沉溺的坑。全专题 17 个原子节点 + 本总览 + README，按 0411 Agent 系统化专题（22 节点、5 轮评议、SABCD≈7.85）的工程化流程产出，目标是达到或超过那条出版线。

---

## §0 序：那堵从没被记录下来的墙

你设计并迭代过至少五个 skill，把记忆从 blocklist 思维翻成 allowlist，用"过拟合 / 泛化"这对机器学习术语给 AI 做过两轮元层校正，还主动把自己 12 个 agent 的架构砍成 5 个——**作为极端 power user，你的使用模式是比任何文献都稀缺的研究材料，却从没被系统记录过。** 人-AI 交互研究最大的缺口，是日志能记录行为却记录不了意图与情境；而这恰恰是你——作为这片田野的**完整成员（Complete Member Researcher）**——花再多钱的外部研究者也买不到的一手数据。

这堵墙的另一面是陷阱：越是无人能证伪"Rick 怎么用 AI"，越容易把"我觉得这 skill 真好用"当成结论写下去。本专题的全部张力就在这里。它的反共识立场是：**自我民族志最大的风险不是"不够客观"，而是"把个人癖好当普遍规律"——而 power user 自我研究把这个风险放大到极致，因为 power user 的整个价值就建立在"我和大多数人不一样"上。** 读完本专题，你能在面试桌 / 选型会 / 复现台上 30 秒说清三件事：(1) 我研究的是自己的**使用方法**而非"AI 领域"，凭什么这是不可复制的求职底料；(2) 一个深度用户的行为拆成哪六层、致命错配发生在哪;(3) 哪些是文件/时间戳可查证的可观察事实，哪些是只有我能填、AI 绝不替我编造的内省。

## §1 专题定位：为什么 0423 配单独建库

按 SHARED_CONTEXT §2 的四条选题判据逐条论证（前 3 条满足 ≥2，第 4 条为真）：

| 判据 | 是否满足 | 论证 |
|---|---|---|
| **中心性**（影响 PM ≥3 个决策链节点） | ✅ | 直接喂养用户研究（A01 把"会用 AI 的 PM"升级为"会研究 AI 使用的 PM"）、产品设计（S01 六层栈本身即需求图谱）、求职定位（R03 信号理论）三条决策链 |
| **误解深度**（业界定义互相矛盾、系统性滑变） | ✅ | "自我民族志"在学界本身就分裂成不可通约的两派——唤起式（Ellis & Bochner）vs 分析式（Anderson 2006），多数人把它误当"反思日记 / N=1 用户研究 / KOL 工作流贴" |
| **速变性**（24 个月内 ≥1 次范式切换） | ⚠️ 部分 | 研究对象本身（Rick 的 AI 使用）不到一年从 chatbot 问答跃迁到多 agent 工厂（G01 五代谱系），是速变性的活体；方法论本身则是稳定经典 |
| **学了就能用** | ✅ | 读完即得六层栈编码骨架（S01）、四轴日志模板（R01）、可直接投递的链信号作品集结构（R03） |

**升高的抽象层**：相对 `04AI/` 已有的单维节点（[Skill 系统的本质](/kb/ai-协作方法论/skill-系统的本质/) 讲 skill 是什么、[AI 记忆过拟合与泛化能力](/kb/基础知识库/ai-记忆过拟合与泛化能力/) 讲一次记忆现象），本专题把这些**散落的产物当作"数据"**，升高一个抽象层——从"分析对象"升到"分析方法"：不再问"skill 好不好用",而问"Rick 设计 skill 的行为本身，作为 power user 的 revealed preference，能反推出什么"。这是它相对所有单维节点不可替代的位置。

> [!warning] 独一无二 ≠ 不需要对手
> 没有第二个人能写"Rick 怎么用 AI"，正如没人能替你做你的内省——这是**资产**。但"无内容层竞品"绝不等于"无方法论对手":Delamont 对自我民族志"navel-gazing"的指控、von Hippel lead user 的代表性质疑、Parasuraman & Manzey 的 automation complacency，是本专题必须正面接住的对手（详见 §6/§7）。本专题的诚实底线是一句话：**所有可观察的（文件/对话/时间戳）如实分析，所有需 Rick 内省的（信任校准、注意力分配、决策动机）一律留 `〔Rick 待填〕` 结构化模板 + 引导问题，绝不替他编造。** 这不是偷懒，是对 Polanyi 默会知识认识论的尊重——代填即"用可言说的废话填充本质上默会的空洞"。

## §3 六模块逐一介绍

### 3.1 概念辨析（A01–A06，6 篇）— 横向"是什么"
**收录**：把"我觉得好用"从轶事转化为可分析数据的全套认识论工序。
- [A01 自我民族志方法论·作为方法非领域](/kb/专题-人文社科透镜/a01-自我民族志方法论-作为方法非领域/)：奠基节点。分析式 vs 唤起式取向声明、"方法非领域"核心论证、四件套判断主轴（轶事冒充数据 / 编造内省 / 反身性流于姿态 / 关系伦理失守）。
- [A02 使用即数据·什么算 observation](/kb/专题-人文社科透镜/a02-使用即数据-什么算-observation/)：可观察性梯度——从行为痕迹到内省判断的分层编码，前置回答"一次使用怎样被'看见'才配称 observation"。
- [A03 信任校准的自我观察](/kb/专题-人文社科透镜/a03-信任校准的自我观察/)：用 Lee & See 2004 框架剖"auto-accept 还是逐行 review diff"这一隐性技能。
- [A04 注意力分配的隐性算法](/kb/专题-人文社科透镜/a04-注意力分配的隐性算法/)：注意力调度框架——AI 不省注意力，只重新分配它，且这套隐性算法是真实瓶颈。
- [A05 Skill 生态建设作为设计实践](/kb/专题-人文社科透镜/a05-skill-生态建设作为设计实践/)：把 skill 设计决策史当 revealed preference——最不需内省、最经得起证伪的一类数据。
- [A06 自我民族志的认识论陷阱](/kb/专题-人文社科透镜/a06-自我民族志的认识论陷阱/)：epistemic audit——偏见 / 反身性 / 代表性（n=1）/ 行为改变四坑。
**何时读**：第一次进专题，或想给任何"我的 AI 使用感受"装上方法纪律时。

### 3.2 代际演化（G01–G02，2 篇）— 纵向"从哪来"（横切）
**收录**：Rick 的 AI 使用从 chatbot 问答 → prompt 工程 → skill 生态 → 记忆系统 → 多 agent 工厂的五代谱系。
- [G01 Rick 的 AI 使用代际演化总图](/kb/专题-人文社科透镜/g01-rick-的-ai-使用代际演化总图/)：反共识主轴——**每次代际跃迁映射的不是模型能力跃迁，而是"把什么外包给 AI"这一委托边界的重划**；用塌缩（12→5 agent）证明边界双向移动、可证伪。
- [G02 使用代际演化详解](/kb/专题-人文社科透镜/g02-使用代际演化详解/)：逐代展开典型工作流 / 能力门槛 / 瓶颈 / 被下代掀翻的接口五件套。
**何时读**：想看时间维度、想反驳"线性进步史"时。

### 3.3 架构剖面（S01–S03，3 篇）— 解剖学"由什么组成"
**收录**：把一个极端 power user 当作"被解剖的系统"。
- ★[S01 深度 AI 用户行为模型剖面](/kb/专题-人文社科透镜/s01-深度-ai-用户行为模型剖面/)：旗舰。**六层委托栈**（intent → delegation → trust-calibration → review-routing → state-externalization → iterative-learning），每层配可观察指标 + 〔Rick 待填〕模板 + PM 含义；判断主轴落三处致命层间耦合（委托×信任→过度依赖 / 审阅×注意力→审阅塌缩 / 外化×学习→记忆过拟合）。
- [S02 使用模式对照矩阵](/kb/专题-人文社科透镜/s02-使用模式对照矩阵/)：模式离散度——power user 的稀缺性不在单一模式极致，而在跨任务的模式切换幅度。
- [S03 个人 AI 工作流系统全景](/kb/专题-人文社科透镜/s03-个人-ai-工作流系统全景/)：把散落的 skill / memory / routines / vault 协议拆开，论证"工作流系统化程度才是真正的 AI 杠杆率"。
**何时读**：要一套可迁移的分层模型、或评估 AI 协作工具时。

### 3.4 实例剖解（E01–E03，3 篇）— 病理学"现实怎么走样"

- ★[E03 专题工厂作为多 Agent 协作 Case 剖解](/kb/专题-人文社科透镜/e03-专题工厂作为多-agent-协作-case-剖解/)：meta 节点。**生产即田野**——正在生产这套笔记的多 agent 流水线本身就是 Rick 编排 AI 的可观察现场；判断主轴五错点 + 五处〔Rick 待填〕。
**何时读**：想看"理论落到真实系统会怎么裂"时；E03 是理解本专题自指结构的钥匙。

**诚实综合分 ≈ 7.85 / 10**（达出版线 ≥7.8）。

> [!warning] 本专题独特性的诚实声明
> 本专题的独特性 = **方法（分析式自我民族志 / CMR）+ 框架（六层委托栈、委托边界、模式离散度、链信号）+ 可观察分析（vault 产物 / 对话存档 / 时间戳 / 本工厂运作）+ Rick 待填的内省留位**，**而非伪装成一份完整的内省**。一个把信任校准、注意力分配、决策动机都"替 Rick 填好"的版本会更"完整"、综合分也许更高看，但那是**虚构**，触犯一票否决项。本专题宁可在"完整度"上扣分，也要守住"留位不代填"——这正是它的认识论价值所在，也是它经得起 Delamont 拷问的根。

### 7.2 三清单

**(a) 业界对手立场显式回应（≥8 处，均落到具体节点）**
1. Delamont「navel-gazing」→ A01 §5 / A06 / E03 §6：接受"纯唤起式不接受外部验证会自我沉溺"，边界"实证标准会丢掉只有 CMR 能触及的意图-情境数据"。
2. Ellis & Bochner「分析式驯化了后现代实践」→ A01 §1 赌注：接受损失第一人称质感，坚持"求职/判断训练库里可迁移性优先"。
3. von Hippel lead user 代表性争议（Franke & Lüthje 2020）→ A01 §2 / G01 §5：接受"前沿性损害可迁移性"，价值定位改为"机制样本而非分布样本"。
4. Lee & See 2004 信任校准 → S01 §3 / A03：信任不可直接观察、只能看外显代理且需当事人自报。
5. Parasuraman & Manzey 2010 complacency → S01 §6 耦合 A / E03 §6 / G01 §5：接受"训练警告消不掉过度依赖"，护栏只是减灾不是免疫。
6. DeepMind Jain et al. 2025（arXiv:2510.26518）→ S01 §3：接受"AI 自报置信度加剧过度依赖"，边界"Rick 信任锚点来自历史协作记忆而非单次置信度"。
7. Bansal et al. 2021（CHI, arXiv:2006.14779）→ S01 §3 / E03 §3：接受"解释提高采纳却没提升人机互补"，在 fleet 倾向"过度增加角色/格式"上同构。
8. Kim 2026（*Consumer Psychology Review*, DOI 10.1002/arcp.70008）→ S01 §5：接受认知卸载有脱技能化风险，边界"Rick 是治理性外化、落在 Skilled Augmentation 象限——但这条边界很脆"。
9. Buçinca et al. 2021（arXiv:2102.09692, N=199）→ S01 §6：cognitive forcing 能降过度依赖但用户最不喜欢、对低动机者无效，代价真实。
10. Spence 1973 信号理论 → R03：论证伪造成本趋零的信号停止传递信息。

**(b) failure scenario 显式标注（≥5 处）**
1. 六层栈假设各层可分离——若 Rick 真实认知高度融合，分层是分析者强加（S01 §7，本模型最大赌注）。
2. N=1 结论无法推广到其他 power user，可能是 Rick 哲学/PM 背景的特例（S01 §7 / A06 / G01）。
3. 可观察产物与内省叙述冲突时（档案显示某 skill 高频触发，但 Rick 回忆"其实不好用、只是手滑"），该信哪个？自我民族志无裁决规则（A01 §3）。
4. write-first 流水线"高吞吐"被误当"高质量"——吞吐与质量正交，吞吐数字最不可信（E03 §4 错点 3）。
5. "委托边界右移 = 能力跃迁"在"委托是被动跟风而非主动判断"场景失效——靠塌缩证据反推主动性，但不保证未来每次扩张都主动（G01 §5）。
6. verisimilitude 被当成 validity——读起来"很真"≠分析可靠，写得动人恰是危险信号（A01 §3）。

**(c) confirmation-bias 砍除（≥5 处）**
1. 早期倾向把 Rick 每个行为都读成"高水平 power user 典范"→ 砍除：power user 不是过度依赖的免疫者，是**高暴露者**（S01 §7，本专题真正的产品洞察反转）。
2. 把"Rick 是 lead user"当纯正面资产 → 砍除：补入"前沿性损害可迁移性"反例（A01 §2）。
3. 早期叙事把每代写成"Rick 越来越牛"→ 砍除：必须并列**收回的委托**（删旧记忆、塌缩 agent），G0→G1 起点不可考、不为叙事完整编造漂亮起点（G01 §5）。
4. 只剖被升格入库的节点（幸存者偏差）→ 砍除：C 级 182 条几乎与 B 级持平，"什么被丢弃、为什么"也是数据（E03 §4 错点 4）。
5. 把 ingestion 沙盒当成"已防住过度依赖"的证据 → 砍除：它可能只是"看起来在防"的姿态，实际审阅仍可能 complacent 放行——留 〔Rick 待填〕 验证（G01 §5 / E03 §5）。
6. 把"自我民族志服务求职"默认为正当 → 砍除：Denzin 批判视角逼问"是否把方法降格成个人品牌工具"（A01 §5，张力保留不强行消解）。
