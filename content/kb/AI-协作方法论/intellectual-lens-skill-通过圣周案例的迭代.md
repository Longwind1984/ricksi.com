# intellectual-lens skill 通过圣周案例的迭代

**核心论点**: Rick 把 Popayán 圣周游行的现场假说（军民共在场建立非正式权力网络 → 反制秦制式军地切割 → 将军不会对人民开枪）丢进 intellectual-lens skill 跑两次，对比另一份 AI 的输出，定位 skill prompt 的薄弱环节并改进——这次迭代的价值在 skill 工程而不是案例分析本身。

## 触发案例：圣周游行的军民假说

Rick 在 Popayán 看圣周游行，提出一个工作假说：

- 游行场域内军方（将军、士兵）与社会其他角色（学校、教会）共在场，构成一种强制的社会"黏着性"
- 这种黏着性建立起秦制几千年来害怕的军地关系 / 非正式权力网络
- 推论：能在圣周游行上屈尊表演的将军和士兵，不会对自己的人民开枪

这个假说本身可作为田野观察的范例：从仪式空间的物理共在场推到军民关系的政治稳定函数，借秦晖的"秦制"分析框架做对照。

## skill 比较的方法

Rick 让 intellectual-lens skill 跑一次，又附上另一个 AI 版本的分析，要求：

- 比较两份分析的异同、优劣
- 思考如何优化 skill prompt

这是把 AI 当作可被工程化的对象——不止是用 AI，是把"AI 输出"作为可比较、可迭代的工件，反向回到提示词层做修补。

## 改动落点：step 2.5 综合环节

迭代结果是 skill 的 step 2.5（综合环节）需要加强，但有边界：

- 强化综合，不能反过来挤压 step 2 的详尽分析
- 综合的输出要点收敛——Rick 直接砍掉"第 3 到第 5 点没必要"，保留前两点

这个截断反映 Rick 对 AI 输出的偏好：**结构要清晰，颗粒度要克制，多即是少**。skill 的 step 数量与每步产出量都是 prompt 工程决策点。

## 可迁移的方法

把单一 skill 运行 → 拿"竞品输出"做对照 → 锁定差距来源在 prompt 的哪一步 → 局部修补——这套流程可以套用到其他 skill：trip-evaluate、doc-coauthoring 等。前提是有"竞品输出"作为参照系，否则单跑容易陷入 self-confirming。

> [!note] 待校验观察
> "圣周游行上屈尊表演的将军不会对人民开枪"这一推论在哥伦比亚军队历史上有反例（如 Palacio de Justicia 事件、社会领袖暗杀）——假说在仪式黏着 vs 制度命令链之间的张力需要更精细的边界条件，不能直接套用到所有军方个体。这是案例本身值得继续展开的方向，但不影响 skill 迭代的方法论收益。
