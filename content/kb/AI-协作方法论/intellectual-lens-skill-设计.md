# intellectual-lens skill 设计

**核心论点**: Rick 把 "随笔写作如何卷入跨学科理论" 抽象成一个 skill 设计需求——非固定模板、主动调用、以最优解释力为优先、兼顾智识边界扩展。这四条原则定义了一个非 SOP 化、信任智能体判断的 skill 形态。

## 1. 需求触发：随笔不是叙事而是现象剖析

Rick 的写作不是以叙事为骨架，而是以社会现象为入口。问题在于：现象层面的观察容易停在 "我观察到 X" 这一步，要往下一层走，需要人类学/社会学/政治学的理论框架去做横切。skill 的功能就是把这种横切动作做成可调用的工具。

> [!quote] Rick 的关键介入
> 我想做一个写作主题概览的skill，帮我构思如何实现。我想实现的效果是因为我的随笔可能不会特别聚焦于叙事，而是会聚焦于一些社会现象的进一步剖析，并且联系人类学、社会学、政治学的理论。我希望这个skill帮我做的就是运用交叉学科的理论框架去看懂社会现实，并且持续拓展我的知识边界。

## 2. 四条设计原则

Rick 在第二轮就把设计约束钉死：

> [!quote] Rick 的关键介入
> 1. 不需要固定输出 模板。
> 2. 由我主动 call skill 触发。
> 3. 理论解析以契合实际场景、最优解释力优先，兼顾智识边界扩展。
> 4. 目前不需要接口。

这四条对应四个否定：
- **否定模板化输出**：拒绝 "结构化呈现"，因为现象的形状决定文章的形状，反过来不行。
- **否定被动调用 / 自动触发**：调用权在 Rick，不在 AI。AI 不能因为 "看起来像理论分析的场景" 就主动启动 skill——主动权属于写作者。
- **否定理论先行的拉扯**：契合场景 > 理论体系完整性。一篇随笔不需要把所有相关理论列一遍。
- **否定接口化封装**：skill 不需要做成独立产品，只需作为写作辅助。

> [!note] 待校验观察
> 第三条 "兼顾智识边界扩展" 与 "最优解释力优先" 之间有张力——前者鼓励 skill 提出 Rick 未必想得到的框架，后者要求收敛到当下场景。在实际调用中，这条张力会决定 skill 是 "更保守地解释" 还是 "更积极地外推"。

## 3. 与 vault 的位置

intellectual-lens 后来被实际 build 出来（见 `20260405-intellectual-lens-skill-通过圣周案例的迭代`）。本对话是它的概念发生学起点——只有原则，未到落地。后续迭代会把这四条原则压成具体的 instruction 文本。
