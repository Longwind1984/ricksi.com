# Polanyi 默会知识与提示工程的认识论张力

**核心命题**：Polanyi 的默会知识（tacit knowledge）命题"We can know more than we can tell" 揭示一个对提示工程具有内在颠覆性的张力——提示工程的努力方向（把意图明言化、形式化）本身就是逻辑实证主义"明言化理想"的当代延伸；而默会维度的存在意味着 prompt 永远无法等价于意图本身。这条张力解释了为什么 Chain-of-Thought 有效、为什么 few-shot 有效、以及为什么 prompt engineering 始终不是"工程"而是"手艺"。

## 一、Polanyi 命题的最小骨架

- **Michael Polanyi（1891-1976）**：匈牙利裔英国物化学家转向科学哲学。1930 年代与布哈林辩论后，对苏联式"计划科学"的认识论危险产生反思。
- 主要著作：*Personal Knowledge*（1958）、*The Tacit Dimension*（1966）。
- 核心句：**"We can know more than we can tell."**

### 近端-远端结构

人类认知总涉及两层：
- **近端项（proximal）**：所*依赖*但不直接注意的细节——肌肉感觉、局部线索、背景信息。
- **远端项（distal）**：所*聚焦*的对象或目标。

骑车时注意力在"前进"（远端），而非每块肌肉发力（近端）。注意力转移到近端反而失去控制。

| 例子 | 近端（默会） | 远端（明言目标） |
|---|---|---|
| 骑自行车 | 平衡微调、重心感知 | 向前行进 |
| 识别人脸 | 五官局部特征 | 认出"是张三" |
| 用工具（锤子） | 锤柄手感、震动 | 钉入钉子 |

工具使用是 Polanyi 最爱的例子：熟练工匠"居住"（dwell in）在工具里，工具成为身体图式的延伸，不再被观察，而是认知的透明中介。

### 直接靶子

Polanyi 矛头指向逻辑实证主义的明言化理想（科学知识可完全形式化）和苏联计划科学（认为可以从中央设计配置科学方向）。其反驳：博士生跟导师学的，大量是无法写入教科书的"品味"——判断哪条结果值得追究、哪个异常是噪音。

## 二、对提示工程的认识论冲击

### 张力一：prompt 永远小于意图

写 prompt 是在指定"远端项"（你想要什么），但模型通过"近端"（统计模式）操作。**你能形式化的部分与你实际想要的部分之间永远存在缺口**——这个缺口就是默会知识所栖居的地方。Prompt engineering 的努力方向（更精确、更结构化、更模板化）本身受这条上限约束。

### 张力二：CoT 为什么有效

Chain-of-Thought 强制把"近端"过程外化为"远端"陈述——把模型先前"穿过"的步骤变成需要被注视的内容。这是用模型自己的语言能力，反向逼近 Polanyi 拒绝的"明言化"。

代价：模型现在注视的内容也是它在生成的内容，**它正在"骑车时盯着自己的腿"**——CoT 在某些任务上反而劣化原本流畅的近端处理。

### 张力三：Few-shot 是默会的搬运

Few-shot 例子起作用，不是因为它们"教"了规则，而是因为它们让模型"看到"了一种风格、一种判断的体感——这种通过样例传递的，正是 Polanyi 意义上的默会维度。一个好提示工程师写的 few-shot examples，本质上是**把自己脑子里的默会判断打包为案例**。

### 张力四：师徒关系的对应物

Polanyi 强调科学训练的核心是师徒制，因为默会知识只能通过共同实践传递。Prompt engineering 的对应物：好 prompt 的设计者通过反复 iterate-and-eye 的过程积累的，是无法写进 prompt 模板库的判断力。**prompt 工程不是工程**——它没有可复制的离散步骤，更像调香师或显微镜下的病理学家的训练。

## 三、规范性推论

对 Rick 这类既懂哲学又做 AI PM 的人，三条推论：

1. **不要追求"完全明言化"**：试图把所有判断写进 prompt 是逻辑实证主义在 LLM 时代的回响。承认存在不可言说的判断空间，把这部分用 few-shot、参考文档、persona 注入处理。
2. **CoT 不是万能放大器**：当任务靠的是流畅的近端模式（直觉性判断、创作性写作、风格匹配），强制 CoT 会破坏它。
3. **评测的默会维度**：判断模型输出好坏的能力本身是默会的——这是为什么 LLM-as-judge 在主观任务上不可靠的根本原因，不是技术问题，是认识论问题。
